Night Training

The single reference for how the app looks and behaves. Build every new feature from these tokens and components and the app stays coherent — like one app made by one team, not a pile of screens. Tokens here map 1:1 to CSS custom properties intended for :root in styles.css.

01 Principles

The feeling: a serious athlete's training app, not medical homework. Dark, focused, fast, and just a little bit cool.

Dark first

A near-black canvas keeps focus on the work and looks good at night when most reps happen. Content sits on subtly raised surfaces, separated by hairline borders rather than heavy shadows.

One loud accent

Volt green means done / progress / go. It is the only saturated color on screen, so it always pulls the eye to what matters. If everything is accent, nothing is.

Big, honest targets

Whole rows are tappable, never tiny checkboxes. Minimum 44px touch height. Feedback is instant and physical (a quick press-scale), because the user is moving, not sitting.

Quiet chrome

Version numbers, hints, and meta text recede into --text-faint. The exercises and today's progress are the only things that shout.

13 Accessibility

Contrast & targets

  • Body text (--text on --bg) ≥ 13:1.
  • Muted text used only for non-essential labels.
  • Every interactive element ≥ 44×44px.
  • Don't rely on color alone — the ✓ glyph also signals done.

Behavior

  • Respect prefers-reduced-motion.
  • Buttons/rows get aria-labels & pressed state.
  • theme-color meta = --bg (#0b0f14).
  • Focus-visible ring in accent for keyboard users.

14 Do & Don't

Do

  • Reach for an existing token before inventing a value.
  • Keep accent rare — done, progress, one primary action.
  • Make the whole row/card tappable.
  • Let chrome (version, hints) stay faint.
  • Bump footer version + CACHE_VERSION together.

Don't

  • Add new brand hues per feature.
  • Use a colored header bar (we went flat/dark).
  • Paint large areas in accent green.
  • Set body copy in uppercase.
  • Use drop shadows to fake hierarchy — use borders.
